# Vacantrix Web — Микросайт

Одностраничный сайт с вкладками «Площадки» и «Приложения», системой входа через email + OTP и административной панелью. Полностью бесплатно: GitHub Pages (фронтенд) + Supabase (база данных и авторизация).

---

## Быстрый старт

### 1. Настройка Supabase

1. Зайдите на [supabase.com](https://supabase.com) → создайте новый проект (бесплатный тариф).
2. Откройте **SQL Editor** → вставьте и выполните содержимое файла `schema.sql`.
3. В **Authentication → Email** включите **Enable Email OTP** и отключите «Confirm email» (если хотите упрощённый вход).  
   Или оставьте включённым — тогда при регистрации пользователь получит письмо-подтверждение.
4. Перейдите в **Project Settings → API** и скопируйте:
   - `Project URL`
   - `anon public` key

### 2. Конфигурация сайта

Откройте файл `js/config.js` и замените:
```js
const SUPABASE_URL  = 'https://ВАШИ_ДАННЫЕ.supabase.co';
const SUPABASE_ANON = 'ВАШИ_ДАННЫЕ_ANON_KEY';
```

### 3. Деплой на GitHub Pages

```bash
git init
git add .
git commit -m "init: vacantrix web"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/vacantrix-web.git
git push -u origin main
```

Затем в GitHub репозитории → **Settings → Pages → Source: Deploy from branch → main / root → Save**.

Сайт будет доступен по адресу `https://ВАШ_ЛОГИН.github.io/vacantrix-web/`.

Обновите в Supabase **Authentication → URL Configuration → Site URL** на этот адрес (для работы email-ссылок).

## Назначение первого администратора

1. Зарегистрируйтесь на сайте через свой email.
2. В Supabase **SQL Editor** выполните (замените email):
```sql
INSERT INTO web_user_roles (user_id, role)
SELECT id, 'admin' FROM auth.users WHERE email = 'ВАШ_EMAIL'
ON CONFLICT (user_id) DO UPDATE SET role = 'admin';
```
3. Войдите на сайт снова — появится кнопка «Панель» в шапке.

## Функциональность

### Главная страница
- **Вкладка «Площадки»** — кнопки-ссылки, сгруппированные по категориям (аккордеон). Управляется из панели.
- **Вкладка «Приложения»** — полная презентация Vacantrix: описание, преимущества, скриншоты (с лайтбоксом), промо-видео, кнопки CTA. При добавлении второго приложения автоматически переходит в режим сетки.

### Авторизация
| Сценарий | Поведение |
|---|---|
| Первый вход (без «Запомнить меня») | email + пароль → OTP на email |
| Вход с «Запомнить меня» | email + пароль, сессия хранится в браузере |
| Повторное открытие (с «Запомнить меня») | автоматический вход без запроса |
| Регистрация | email + пароль → письмо-подтверждение |

### Административная панель (`/admin.html`)
- Управление площадками: добавление, редактирование, скрытие, удаление, drag-and-drop сортировка
- Управление приложениями: все поля (описание, скриншоты, ссылки, видео, статус), drag-and-drop
- Просмотр пользователей, назначение администратора
- Смена пароля

## Добавление нового приложения

1. Войдите на сайт → «Панель» → вкладка «Приложения» → «+ Добавить».
2. Заполните поля (название, slug, описание, скриншоты — URL на строку, преимущества — текст на строку).
3. Нажмите «Сохранить» — приложение сразу появится на сайте.

При наличии **одного** приложения — полноэкранная презентация.  
При **двух и более** — сетка карточек.

## Используемый стек

| Компонент | Сервис | Тариф |
|---|---|---|
| Фронтенд | GitHub Pages | Бесплатно |
| База данных | Supabase (PostgreSQL) | Бесплатно (Spark) |
| Авторизация + Email OTP | Supabase Auth | Бесплатно |
| CDN Supabase JS | jsDelivr | Бесплатно |

Никаких платных зависимостей. Сайт работает автономно без локального сервера.
